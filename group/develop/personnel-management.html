应用端开发人员需实现成员管理相关接口，与SID进行人员同步。

查询成员

查询某个人员信息。 SID调用“创建成员”、“更新成员”、“删除成员”等接口时，会先调用该接口，执行查询操作，提高群组推送的容错性。

请求⽅式：GET（HTTPS或HTTP）

请求地址：http://{server}/app/user/get/{id}

注：https方式类似。

请求头:

header: Signature (通过MD5获取到的签名值)

header: Timestamp（当前时间戳）

参数说明：

参数 是否必须 说明
id 是 学号或工号

返回结果：

{
    "errcode":200,
    "errmsg":"ok",
    "userInfo":{
        "userId": "YT001",
        "name": "xxx",
        "xb": "1",
        "tel": "18732990023",
        "sflbdm":"01",
        "departments":["xxxx1","xxxx2"]
    }
}

参数说明：

参数 是否必须 说明
errcode 是 返回码，成功为200
errmsg 是 对返回码的文本描述内容
userInfo 是 用户信息
userId 是 学号或⼯号
name 是 用户姓名
xb 否 用户性别:1-⼥，2-男，3-不知
tel 否 ⽤户绑定手机号
sflbdm 是 ⽤户身份类别代码
departments 是 ⽤户所属部⻔ID列表
